# 템플릿: tutorial-stepflow
서사: 훅(제품 소개) → Step 1..N(제품 사용 단계, 자막 "Step 1 — ...") → 결과 + CTA.

## 핵심 원칙: 고정 세션
- **같은 세션** = 장소·인물·옷·조명을 모든 step에서 고정. 손동작·단계만 변함.
- day-progression(날마다 장소·패션·포즈 변주)과 **반대**. 같은 세션으로 보여야 사용법이 연속적으로 읽힘.
- 고정(절대): 인물·피부톤·의상·배경·조명·카메라 거리·프레이밍.
- 변하는 것: 손 위치·동작·제품 상태(짜기→바르기→흡수 등).

## 컷신 패턴(synopsis 생성 규칙)
- hook: 제품 들고 카메라에 소개(또는 짧은 훅 자막). start-only. 자막=훅 문구.
- stepN(N=1..최대 5): 단계별 사용 동작. start-only. 자막=`Step N — <행동>`.
- result: 사용 후 피부/결과 리빌. start-only 푸시인. 자막=결과 문구.
- cta: 마지막 모션에 CTA 동작 결합(또는 엔드카드 대안). 자막=CTA 문구.

## 모델/설정
- 이미지: nano_banana_2(9:16, 2k). 자막/보드 생성 강조 시 gpt_image_2 대안 가능.
- 영상: kling3_0(pro, 9:16, 5s, 무음).

## 인물 일관성 절차
1. **얼굴 나오는 UGC**: ad-persona 서브에이전트 → Soul Character 학습(`higgsfield-soul-id`) → `reference_id` 확보 → 이후 모든 컷에 `--soul-id <ref_id>` 적용(모델: `text2image_soul_v2` 또는 `soul_cinema_studio`).
2. **무얼굴 바디형**: 마스터 레퍼런스 1장 확정 → step별 변주 시 `--image` 레퍼런스.

## 스토리보드 보드 기법 (권장)
- Step 1~4를 **한 장의 와이드 이미지(4분할 보드)**로 동시 생성: 공유 배경·조명·인물, 각 슬롯에 Step 자막 오버레이.
- 생성 후 9:16 슬롯별로 크롭 분할 → 컷 간 일관성 구조적 보장.
- 대안: 마스터 기준 step별 독립 생성(각 step에 `--image` 마스터 레퍼런스).

## 가드레일 (필수)
guardrails/common.md를 따른다. 이 템플릿에서 특히:
- NSFW: 착장(의상·데님 등) 명시, "bare" 금지, 증상 텍스트 묘사, 차단 시 레퍼런스 없이 통과 후 변주.
- 게이트: 페르소나(Soul ID 또는 마스터) 확정 서브게이트 → 보드/훅 컷 확인 게이트1 → 전체 클립 게이트2 → 최종 게이트3.
- 제형·거동 매칭: 제품 제형컷 기반 텍스처(되직·흐름·알갱이 등) 정확히 반영.
- ad-qa 자체검수: 각 이미지·클립 생성 후 검수.
