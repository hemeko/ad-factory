# 템플릿: ugc-day-progression
서사: 인트로(제품 짜기) → Day1/3/5/7(롤링+고마쥬, 단계적 개선) → 매끈·물광 페이오프 → 엔드카드.

## 컷신 패턴(synopsis 생성 규칙)
- intro: 부위에 제품 짜는 컷. start-only. 자막=훅.
- dayN(N∈brief.days): 롤링+고마쥬. start-only(연속롤링). 자막="DAY N". 부위·범위 day마다 변주.
- payoff: 마지막 day 매끈·물광 리빌. start-only 푸시인. 자막=마무리.
- endcard: 제품컷 + 브랜드/제품명.

## 모델/설정
- 이미지 nano_banana_2(9:16,2k). 영상 kling3_0(pro,9:16,5s,무음).

## 일관성 절차
1) 마스터 베이스(인물+부위+포즈+장소): 구도 레퍼런스 + 증상/제형 레퍼런스로 생성.
2) Day 리빌 = 마스터 레퍼런스로 피부 상태만 변주(+장소/조명).
3) 액션(고마쥬/짜기) = 해당 day 리빌 기반 + 텍스처 레퍼런스로 손/제형/고마쥬 추가.

## 일관성 vs 시간경과 변주 (중요)
- 고정(절대 유지): 인물·피부톤·부위·얼굴유무(마스터 기준).
- Day별 변주(필수): 장소·패션(하의/의상)·**포즈/다리자세**·조명을 Day마다 다르게. 같은 장소·옷·자세로 고정하면 "같은 날"처럼 보여 시간경과 컨셉이 깨짐 — 일관성을 과하게 잡지 말 것.
- 피부 개선 곡선은 단계적.

## 개선 곡선(기본, brief에서 조정)
Day1 중간 → Day3 눈에띄게 → Day5 거의매끈 → Day7 무결점+물광.
